# Step 3 — "Age" — Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Build the standalone, controlled-only `Age` step (Step 3 of the 14-step skin-cancer questionnaire): a numeric age input with an explicit Continue button and a calm under-18 redirect.

**Architecture:** A single Preact function component `Age` composed from `@hirslanden/ds` primitives, mirroring the shipped `PreviousDiagnosis` step. It owns no answer state — a numeric `value` prop drives the input, and derived booleans drive a disable-until-valid Continue button (azure DS `primary`/`lg`) that is swapped for a dashed `InfoCard` when a valid age is under 18. A wrapping `<form>` provides Enter-to-submit. All copy flows through the i18n layer.

**Tech Stack:** Preact 10, TypeScript 6, `@hirslanden/ds` (React-authored, consumed via `preact/compat`), Tailwind v4 + `--hds-*` design tokens, Vitest + `@testing-library/preact`, Storybook 10, Biome.

**Spec:** `docs/superpowers/specs/2026-06-26-age-step-design.md` (read it before starting).

## Global Constraints

Every task implicitly includes these (exact values from the spec):

- **No git.** Never `commit`, `push`, or `branch` — the maintainer owns all git. Each task ends by running verification and leaving changes in the working tree (app **and** the one DS fix in Task 4).
- **Preact 10.** Import hooks from `preact/hooks`; no React import (JSX is `jsxImportSource: "preact"`). Use **`class`** on native elements, **`className`** on DS components.
- **Controlled-only.** No component-local answer `useState`, no `defaultValue`. The numeric `value` prop drives the input text and all derived UI state.
- **Imports.** Intra-app via the `@root` alias (`@root/src/...`); DS imports stay package-scoped (`@hirslanden/ds/<subpath>`). Biome's `organizeImports` is enforced — keep imports alphabetical (the `biome check --write` step in each task does this for you).
- **i18n.** All user-facing text via `translate('<string-literal-key>')`; keys live in all three of `src/locale/{en,de,fr}.json`; run `pnpm gen:i18n` after any locale edit; never hand-edit `src/i18n/generated.ts`.
- **Biome.** Single quotes, **semicolons always**, brace every control statement, no `any` (use `unknown` + guards), **no `as`** (widen types instead), no conditional object spreads. `pnpm lint` = `biome check .` (check-only), so each task runs `biome check --write` on the touched paths first.
- **Styling.** Tailwind + `--hds-*` token-backed utilities only (no hardcoded brand hex/px/font); mobile-first.
- **Bounds.** `MIN_AGE = 0`, `MAX_AGE = 120`, `ADULT_AGE = 18` — valid ages are **whole years 0–120**.
- **A11y.** WCAG 2.1 AA; the gate is **zero axe AA violations except the accepted brand-contrast deviation** (muted `#938880` 3.45:1, azure `#0094d4` 3.39:1 incl. the white-on-azure CTA).
- **DS edits are in-scope when forced.** Importing a DS primitive compiles its *source* into the app's program, which can surface latent DS type bugs (recorded project reality). Task 4 fixes exactly one such bug in `@hirslanden/ds` `InfoCard`; do not otherwise modify DS. Flag a `TODO(ds)` at the `ArrowRightIcon` call site (Button has no trailing-icon slot — don't add a variant).

All app commands run from `apps/skin-cancer-screening`; the DS-package commands in Task 4 note their own working dir.

- [ ] **Step 4: Create the component (static shell + input + Back)**

`src/components/steps/Age/index.tsx`:

```tsx
import { Heading } from '@hirslanden/ds/heading';
import { ArrowLeftIcon } from '@hirslanden/ds/icon';
import { Input } from '@hirslanden/ds/input';
import { Overline } from '@hirslanden/ds/overline';
import { StepProgress } from '@hirslanden/ds/stepprogress';
import { Text } from '@hirslanden/ds/text';
import { TextLink } from '@hirslanden/ds/textlink';
import type { AgeProps } from '@root/src/components/steps/Age/Age.types';
import { useTranslation } from '@root/src/i18n';

const DEFAULT_CURRENT = 3;
const DEFAULT_TOTAL = 14;

export function Age({
  value,
  onChange,
  onBack,
  current = DEFAULT_CURRENT,
  total = DEFAULT_TOTAL,
}: AgeProps) {
  const { translate } = useTranslation();

  const hasAge = typeof value === 'number' && Number.isFinite(value);

  return (
    <section data-testid="age" class="mx-auto max-w-2xl border border-border bg-bg p-6 sm:p-8">
      <StepProgress
        current={current}
        total={total}
        label={translate('general.progress.stepOf', { current, total })}
      />

      <div class="mt-6" data-testid="age-overline">
        <Overline>{translate('steps.age.overline')}</Overline>
      </div>

      <div class="mt-2" data-testid="age-question">
        <Heading level={1} className="text-xl!">
          {translate('steps.age.question')}
        </Heading>
      </div>

      <div class="mt-2" data-testid="age-description">
        <Text tone="muted">{translate('steps.age.description')}</Text>
      </div>

      <form
        noValidate
        data-testid="age-form"
        onSubmit={(event) => {
          event.preventDefault();
        }}
      >
        <div class="mt-6" data-testid="age-field">
          <Input
            label={translate('steps.age.label')}
            type="number"
            inputMode="numeric"
            min={0}
            max={120}
            step={1}
            value={hasAge ? String(value) : ''}
            onInput={(event) => {
              const raw = event.currentTarget.value;
              if (raw === '') {
                onChange?.(undefined);
                return;
              }
              const parsed = Number(raw);
              onChange?.(Number.isNaN(parsed) ? undefined : parsed);
            }}
            data-testid="age-input"
            className="w-24"
          />
        </div>

        <div class="mt-8 flex min-h-32 items-start justify-between gap-4" data-testid="age-actions">
          <TextLink leadingIcon={<ArrowLeftIcon />} onClick={onBack}>
            {translate('steps.age.back')}
          </TextLink>
        </div>
      </form>
    </section>
  );
}
```

> Note: `min`/`max` are inline literals here for the bare input; Task 3 introduces the `MIN_AGE`/`MAX_AGE`/`ADULT_AGE` constants and rewires `min`/`max` to them alongside the validity logic. If `tsc` rejects `noValidate` on the `<form>`, switch it to the lowercase HTML attribute `novalidate`.

The fix widens `InfoCardProps` to `HTMLAttributes<HTMLElement>` (matching `Card`). Steps 5–7 below intentionally reproduce the failure before fixing it.
