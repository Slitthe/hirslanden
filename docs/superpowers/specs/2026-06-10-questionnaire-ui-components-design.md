# Hirslanden DS — Questionnaire UI Components Design Spec

**Date:** 2026-06-10
**Status:** Approved (decisions locked)
**Repo:** `hirslanden-ds`
**Builds on:** `docs/superpowers/specs/2026-06-09-hirslanden-ds-component-library-design.md`

## 1. Purpose & Goals

Add the UI components needed to build a **skin-cancer risk-orientation widget** — a client-only, 14-step questionnaire (intro → warning signs → demographics → skin/sun/mole history → results → FAQ) that stores nothing and ends with two risk profiles (melanoma + BCC/SCC) plus a "get in touch" CTA. The widget itself (flow logic, state machine, scoring) is **out of scope** here; this spec covers only the **reusable UI component library** behind it.

This work is **purely additive** to the existing `@hirslanden/ds` library. It introduces no new tooling, build steps, or architectural decisions — every component drops into the conventions already locked in the base spec.

### Source material
The component inventory is derived from seven annotated screenshots (`steps 1–4, 8–10`) at `C:\Users\sil\Desktop\hirslanden-screenshots`. Only a representative subset of the 14 steps was provided; steps with repeating UI were omitted intentionally. The orange numbered circles (1/2/3) overlaid on the screenshots are **annotation markers** pointing at components — they are NOT part of the UI and produce no component.

### Design fidelity
Visual design is **not final** — the consumer will restyle later. Priority is **reusable, accessible, well-structured components** that match the existing library's conventions, not pixel-perfect fidelity. Use the screenshot design as the starting visual reference only.

## 2. Inherited Decisions (from base spec — unchanged)

All authoring, build, packaging, and testing decisions from the 2026-06-09 spec apply verbatim. The load-bearing ones for this work:

| Area | Decision |
|---|---|
| Authoring | Plain **React 18 API**, automatic JSX runtime, `forwardRef`, `useId` for all a11y ids, **no `defaultProps`** (default params), no React-19-only APIs, library never imports `preact`. |
| Styling | Co-located **CSS Modules** per component; tokens as `var(--hds-…, fallback)` so a standalone-imported component never renders unstyled. |
| Per-component folder | `Name.tsx`, `Name.module.css`, `Name.test.tsx`, `Name.stories.tsx`, `index.ts`. Root `src/index.ts` is a pure re-export barrel. |
| Packaging | ESM-only, per-component subpath exports generated by `scripts/generate-exports.mjs`, `sideEffects: ["**/*.css"]`, React as peerDeps. |
| Tree-shaking | Importing one component must not pull an **unrelated** component's JS/CSS. Deliberate internal composition (e.g. `InfoCard` → `Card`) is fine — it only pulls styles actually used. |
| Testing | Dual-project Vitest (React + `preact/compat` alias); every component's tests must pass in **both** projects. Plus `publint`/`attw` and the Preact smoke app. |
| Storybook | v10 `@storybook/react-vite`, one `*.stories.tsx` per component. |

## 3. Composition Strategy (decided)

**Standalone components with deliberate internal composition.** Each component is its own self-contained folder. A few build on a shared base:

- `InfoCard` and `Callout` build on **`Card`**.
- `TextLink` and `Accordion` use **`Icon`**.
- `OptionCard` consumes optional context from **`OptionGroup`**.
- `Disclaimer` is **not** a component — it is the documented usage `Text size="sm" tone="muted"`.

**Paired components share one folder and one subpath export.** `OptionCard` + `OptionGroup` live in `src/components/OptionGroup/` (folder barrel re-exports both → subpath `@hirslanden/ds/optiongroup`); `Accordion` + `AccordionItem` live in `src/components/Accordion/` (→ `@hirslanden/ds/accordion`). This matches `scripts/generate-exports.mjs`, which emits one lowercased subpath per component folder. All other components get their own folder/subpath as usual.

*Rejected alternatives:* (B) fully flat — duplicates border/surface CSS across components, invites drift; (C) one polymorphic `Box`/`Text` mega-primitive — fewer files but fights the "named component per element" goal.

## 4. Component Set (14)

Each entry lists the public API surface and key behaviour. Props extend the relevant native element attributes unless noted. All components accept `className` (merged) and forward refs to their root element where a ref is meaningful.

### 4.1 Typography & primitives

#### `Overline`
Uppercase, letter-spaced section label (`WARNING SIGNS`, `YOUR RESULTS`, the step-1 meta line).
- Props: `as?: 'span' | 'p' | 'div'` (default `'span'`), plus native attrs, `children`.
- Renders styled element with `text-transform: uppercase` and `var(--hds-letter-spacing-wide)`.

#### `Heading`
Page/question titles.
- Props: `level: 1 | 2 | 3` (required) → renders `h1`/`h2`/`h3`; plus native heading attrs, `children`.
- Visual size is driven by `level` via tokens; semantic level and visual size stay coupled (consumers needing a mismatch override via `className`).

#### `Text`
Body copy. **Disclaimer/fine-print is `Text size="sm" tone="muted"`.**
- Props: `as?: 'p' | 'span'` (default `'p'`), `tone?: 'default' | 'muted'` (default `'default'`), `size?: 'sm' | 'md'` (default `'md'`); native attrs, `children`.

#### `Icon`
Small built-in named SVG set. Single module (all paths live together — importing `Icon` pulls the whole small set, which is acceptable).
- Props: `name: 'plus' | 'minus' | 'arrow-left' | 'arrow-right' | 'arrow-down'` (required), `size?: number | string` (default `'1em'`), `title?: string`; spreads to `<svg>`.
- Decorative by default (`aria-hidden`); when `title` is provided, exposes `role="img"` + `<title>`.

#### `Divider`
Styled horizontal rule.
- Props: native `<hr>` attrs. Renders `<hr role="separator">`.

#### `List`
Bulleted list (the "What contributed" lists).
- Props: `children` (`<li>` elements); native `<ul>` attrs. Styled `<ul>`. A `List.Item` convenience export wrapping `<li>` is optional; default is plain `<li>` children.

### 4.2 Surfaces

#### `Card`
Base bordered surface — used as the widget frame **and** as content cards.
- Props: `as?: ElementType` (default `'div'`), `variant?: 'solid' | 'dashed'` (default `'solid'`), `padding?: 'none' | 'sm' | 'md'` (default `'md'`); native attrs, `children`.

#### `InfoCard`
Dashed card with an overline title + body — the 3-up `PRIVATE` / `FOR ADULTS` / `NO PHOTOS` grid. Builds on `Card` (`variant="dashed"`) + `Overline`.
- Props: `label: string` (the overline), `children` (body); native attrs.

#### `Callout`
Emphasis message box.
- Props: `variant?: 'accent' | 'subtle'` (default `'accent'`), `action?: ReactNode` (optional trailing action slot, e.g. a Button), `children`; native attrs.
- `accent` = solid subtle background with a left accent bar ("What's driving the recommendation"). `subtle` = dashed border ("Have questions?", "Want to check your moles?" + ABCDE action).

### 4.3 Navigation

#### `TextLink`
Underlined link/action with optional icon. "← Back" and "Start over" are actions, not URLs.
- Props: `href?: string` (renders `<a>` when present, else `<button type="button">`), `leadingIcon?: ReactNode`, `trailingIcon?: ReactNode`, `children`; native attrs of the resolved element.
- forwardRef to the resolved element.

### 4.4 Composite widgets

#### `StepProgress`
"Step X of 14" label + progress track.
- Props: `current: number`, `total: number`, `label?: string` (default `"Step {current} of {total}"`); native attrs on wrapper.
- Renders the label + a `role="progressbar"` element with `aria-valuenow={current}`, `aria-valuemin={0}`, `aria-valuemax={total}`, `aria-valuetext={label}`. Fill width = `current / total`.

#### `RiskScale`
Segmented, labeled risk meter (two appear side-by-side in results).
- Props: `title: string` (e.g. `"Melanoma"`), `value: string` (e.g. `"Low"`), `segments: string[]` (e.g. `['Low','Slightly','Elevated','Significant']`), `activeIndex: number`; native attrs.
- Renders title + value (value aligned end), a row of segments with the segment at `activeIndex` filled, and each segment's label beneath. Accessible label combines title + value (`aria-label={`${title}: ${value}`}` on the meter region).

#### `OptionCard` + `OptionGroup`
Selectable choice cards. Three observed forms: title-only, title + description, title + description + leading thumbnail.
- **`OptionCard`** props: `value: string`, `title: ReactNode`, `description?: ReactNode`, `media?: ReactNode` (leading thumbnail slot), `selected?: boolean` (controlled when standalone), `disabled?: boolean`, plus `onClick`/native button attrs.
  - Renders a `<button>` (or `role="radio"` element inside a group). States: default / hover / focus-visible / selected / disabled, expressed via `data-selected` + CSS.
  - When rendered inside an `OptionGroup`, it reads `selected` from context (matching `value`) and reports selection via context — the standalone `selected` prop is then ignored.
- **`OptionGroup`** props: `value?: string`, `defaultValue?: string`, `onChange?: (value: string) => void`, `label?: string` (for `aria-label`), `children` (OptionCards).
  - Renders `role="radiogroup"`. Manages selection (controlled or uncontrolled). Each child OptionCard gets `role="radio"` + `aria-checked`. Implements **roving tabindex** with arrow-key navigation (Up/Left → previous, Down/Right → next, wrapping), Home/End, and Space/Enter to select. Single-select only (matches all observed steps).
  - Context shape: `{ value, select(value), register/focus helpers }`.

#### `Accordion` + `AccordionItem`
FAQ rows with `+`/`−` toggle.
- **`Accordion`** props: `allowMultiple?: boolean` (default `true`), `defaultOpen?: string[]`, `value?: string[]`, `onChange?: (openIds: string[]) => void`, `children`.
- **`AccordionItem`** props: `value: string` (stable id), `title: ReactNode`, `children` (panel content), `disabled?: boolean`.
  - Header is a `<button aria-expanded aria-controls>`; panel is a region (`role="region" aria-labelledby`) toggled via `hidden`. `+`/`−` rendered via `Icon`. Ids via `useId`. Open state lives in `Accordion` (controlled or uncontrolled); `allowMultiple={false}` makes it single-open.

## 5. Design Tokens (extend `src/styles/tokens.css`)

Add tokens (keep existing ones). Names follow the `--hds-*` convention. Components reference them as `var(--hds-…, <fallback>)`.

- **Color:** `--hds-color-muted` (~`#6b7280`), `--hds-color-strong` (near-black, ~`#1a1a1a`), `--hds-color-track` (progress/segment background, ~`#e5e7eb`), `--hds-color-surface-subtle` (callout bg, ~`#f5f5f5`), `--hds-color-border-dashed` (~`#c7ccd1`), `--hds-color-accent-bar` (Callout accent left bar, ~`#1a1a1a`).
- **Typography:** font-size scale `--hds-font-size-{xs,sm,md,lg,xl,2xl}`; `--hds-font-weight-{regular,medium,bold}`; `--hds-letter-spacing-wide` (Overline); `--hds-line-height-{tight,normal}`.
- **Spacing:** extend with `--hds-space-{4,5,6}` as needed.
- **Layout:** `--hds-radius-sm`; optionally `--hds-card-max-width` for the widget frame.

> The active progress/segment fill and primary buttons in the screenshots are near-black, whereas the existing `--hds-color-primary` is blue. Tokens introduced here use neutral/near-black values to match the screenshots; **design is not final**, so these are starting values the consumer will retheme.

## 6. Build Order (dependency-aware)

1. Extend `src/styles/tokens.css`.
2. **Leaf primitives** (no component deps): `Icon`, `Overline`, `Heading`, `Text`, `Divider`, `List`.
3. **`Card`** → then `InfoCard` (Card + Overline), `Callout` (Card-based).
4. **`TextLink`** (uses `Icon`).
5. **`StepProgress`**.
6. **`RiskScale`**.
7. **`OptionCard` + `OptionGroup`** (context, roving tabindex).
8. **`Accordion` + `AccordionItem`** (uses `Icon`, context).

Each component, in order: write failing test → CSS module → `.tsx` → folder `index.ts` → add to root barrel → `npm run gen:exports` → `npm run typecheck` → `npm test` (both projects green) → story → commit.

## 8. Accessibility Notes

- `OptionGroup` = `radiogroup`, children = `radio`, roving tabindex + arrow keys (per WAI-ARIA radio group pattern).
- `Accordion` headers are real `<button aria-expanded aria-controls>`; panels are labelled regions; `Icon` toggles are decorative (`aria-hidden`).
- `StepProgress` = `progressbar` with aria value attributes + `aria-valuetext`.
- `RiskScale` exposes a combined `aria-label`; segment labels are visible text.
- `TextLink` renders a real `<a>` (with `href`) or `<button>` (action) — never a clickable `<div>`.
- All ids generated via `useId`; all interactive elements reach focus-visible styling.
- `@storybook/addon-a11y` is available to spot regressions during development.

## 9. Out of Scope

- The questionnaire **flow/state machine, scoring, and embedding** — components only.
- A dedicated `Disclaimer` component (it is `Text size="sm" tone="muted"`).
- A `Badge` component (the orange circles are annotations).
- Multi-select option groups, multi-theme/dark mode, SSR, publish/release automation (all deferred per base spec).
- Pixel-perfect visual fidelity (design is not final).

## 10. Risks & Mitigations

| Risk | Mitigation |
|---|---|
| Roving-tabindex / arrow-key nav behaves differently under `preact/compat` | Cover keyboard nav in `OptionGroup` tests; both Vitest projects must pass. |
| Accordion panel toggle relies on focus/portal quirks | No portals used; plain `hidden` toggle + labelled region; tested in both projects. |
| Internal composition (`InfoCard`→`Card`) accidentally bloats an unrelated import | `verify:treeshake` guards unrelated leakage; composition deps are intentional and used. |
| New tokens missing fallbacks → unstyled standalone component | Every component uses `var(--hds-…, <fallback>)`; spot-check by rendering a component story without importing tokens. |
| Token color mismatch (blue primary vs near-black screenshots) | Documented as non-final starting values; consumer rethemes via tokens. |
